# LightsOut

[한국어](./README.md) | [日本語](./README.jp.md)

Unity ベースの Android 向け 2D パズルゲームです。  
古典的なパズルゲームである **Lights Out** をベースに、決められたステージをクリアしていく基本モードと、ランダムなパズルを継続的に解いていく無限モードを実装しました。  
特に無限モードでは、ランダムに生成されたパズルが解けるかどうかを判定するため、線形代数に基づく行列演算とガウス消去法を活用した解の検証アルゴリズムを実装しました。

## 開発意図

このプロジェクトは、単にパズルゲームを実装するだけでなく、  
ゲームのルールを数学的に分析し、コード上で検証する経験を目的として制作しました。

特に無限モードでは、ランダム生成コンテンツの品質を保証するため、  
Lights Out パズルのルールを線形方程式として表現し、  
ガウス消去法を用いて解けるパズルのみを生成するように実装しました。

また、ステージデータ管理、セーブシステム、多言語対応、テストコードもあわせて構成し、  
小規模なゲームであっても、実際にサービス可能な形に近づけることを目標としました。

---

## ゲームルール

LightsOut は、すべてのライトを消すことを目標とするパズルゲームです。

プレイヤーが1つのブロックを押すと、次の位置の状態が反転します。

- 選択したブロック
- 上のブロック
- 下のブロック
- 左のブロック
- 右のブロック

すべてのブロックが消灯すると、ステージクリアとなります。

## 主な機能

### 1. 基本ステージモード

CSV ファイルで管理される、全 **27 ステージ**を提供します。

| 難易度区間 | ボードサイズ | ステージ数 |
|---|---:|---:|
| 序盤 | 3x3 | 9個 |
| 中盤 | 5x5 | 9個 |
| 終盤 | 7x7 | 9個 |

ステージをクリアすると次のステージが解放され、進行状況は `PlayerPrefs` を通じて保存されます。

### 2. 無限モード

無限モードでは、ランダムなパズルを生成します。

単純にランダムなボードを生成すると解けないパズルが出現する可能性があるため、  
生成されたパズルを解の検証アルゴリズムで確認し、解けるパズルのみをゲームに使用します。

### 3. 解の検証アルゴリズム

無限モードの中核となるのは、**ランダムに生成されたパズルが実際に解けるかどうかを判定すること**です。

Lights Out パズルでは、各ブロックを押すか押さないかを `0` または `1` で表すことができ、  
各入力が周囲のブロックに与える影響を行列として表現できます。

このプロジェクトでは、次の手順でパズルの可解性を判定します。

```text
現在のボード状態を入力
        ↓
各ボタンが影響を与える位置を基準に係数行列を生成
        ↓
現在のボード状態をベクトルに変換
        ↓
拡大行列を生成
        ↓
GF(2) 上でガウス消去法を実行
        ↓
矛盾行の有無を確認
        ↓
解ける / 解けないを判定
```

### アルゴリズム適用理由

無限モードで単純なランダム生成のみを使用した場合、  
プレイヤーがどれだけ操作しても解けないパズルが生成される可能性があります。

これを防ぐため、生成されたボードに対して事前に解の検証を行い、  
解けるボードのみを実際のプレイに使用するように構成しました。

## データ管理

### ステージデータ

ステージ情報は CSV ファイルで管理します。

```text
1,,,,,,
1,1,0,,,,
1,0,1,,,,
0,1,1,,,,
```

各値は次の意味を持ちます。

| 値 | 意味 |
| --- | --- |
| `0` | 消灯しているブロック |
| `1` | 点灯しているブロック |

ステージデータをコードから分離して管理することで、  
新しいステージの追加や難易度調整を行う際に、コードを変更せず CSV のみを修正できるようにしました。

---

### プレイヤーデータ保存

プレイヤーの進行状況と設定値は `PlayerPrefs` を通じて保存します。

| データ | 説明 |
| --- | --- |
| 選択中のステージ | 現在プレイするステージ |
| 解放済みステージ | クリア進行状況 |
| 現在スコア | 無限モードの現在スコア |
| 最高スコア | 無限モードの最高記録 |
| BGM ミュート | BGM のミュート有無 |
| SFX ミュート | 効果音のミュート有無 |
| BGM 音量 | BGM の音量 |
| SFX 音量 | 効果音の音量 |

## 技術的な工夫

### 1. 解けるランダムパズルの生成

無限モードで最も重視した点は、  
ランダムに生成されたパズルがプレイヤーにとって不公平にならないようにすることでした。

単純なランダムボードでは解けない状態が生成される可能性があるため、  
パズルを行列に変換し、ガウス消去法を適用して解が存在するかどうかを検証しました。

これにより、無限モードでも常に解けるパズルだけを提供できるようにしました。

---

### 2. ステージデータとコードの分離

ステージ情報をコード内に直接記述すると、ステージを修正するたびにコードを変更する必要があります。

この問題を改善するため、ステージ情報を CSV ファイルとして分離しました。

この方式のメリットは次のとおりです。

- コードを修正せずにステージを追加可能
- 難易度調整が容易
- データ解析ロジックを個別にテスト可能
- ゲームロジックとコンテンツデータを分離可能

---

### 3. テスト可能な構造作り

Unity では、多くの機能が `MonoBehaviour` やシーン上のオブジェクトに依存しやすくなります。

しかし、すべてのロジックが `MonoBehaviour` に結び付いているとテストが難しくなるため、  
可能な限りロジックを通常の C# クラスとして分離しました。

例えば、`LightsOutSolver`、`LevelLoader`、`DataService` は、  
ゲームオブジェクトがなくてもテストできるように構成しました。
