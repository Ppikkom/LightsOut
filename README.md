# LightsOut

[한국어](./README.md) | [日本語](./README.jp.md)

Unity 기반 Android용 2D 퍼즐 게임입니다.  
고전 퍼즐 게임인 **Lights Out**을 기반으로, 정해진 스테이지를 클리어하는 기본 모드와 무작위 퍼즐을 계속 해결하는 무한 모드를 구현했습니다.
특히 무한 모드에서는 무작위로 생성된 퍼즐의 해결 가능 여부를 판별하기 위해,
선형대수 기반의 행렬 연산과 가우스 소거법을 활용한 해 검증 알고리즘을 구현했습니다.

## 개발 의도

이 프로젝트는 단순히 퍼즐 게임을 구현하는 것에서 끝나지 않고,  
게임 규칙을 수학적으로 분석하고 코드로 검증하는 경험을 목표로 제작했습니다.

특히 무한 모드에서는 랜덤 생성 콘텐츠의 품질을 보장하기 위해  
Lights Out 퍼즐의 규칙을 선형 방정식으로 표현하고,  
가우스 소거법을 활용해 해결 가능한 퍼즐만 생성하도록 구현했습니다.

또한 스테이지 데이터 관리, 저장 시스템, 다국어 지원, 테스트 코드를 함께 구성하여  
작은 규모의 게임이라도 실제 서비스 가능한 형태에 가깝게 만드는 것을 목표로 했습니다.

---

## 게임 규칙

LightsOut은 모든 불을 끄는 것을 목표로 하는 퍼즐 게임입니다.

플레이어가 하나의 블록을 누르면 다음 위치의 상태가 반전됩니다.

- 선택한 블록
- 위쪽 블록
- 아래쪽 블록
- 왼쪽 블록
- 오른쪽 블록

모든 블록이 꺼지면 스테이지가 클리어됩니다.

---

## 주요 기능

### 1. 기본 스테이지 모드

CSV 파일로 관리되는 총 **27개 스테이지**를 제공합니다.

| 난이도 구간 | 보드 크기 | 스테이지 수 |
|---|---:|---:|
| 초반 | 3x3 | 9개 |
| 중반 | 5x5 | 9개 |
| 후반 | 7x7 | 9개 |

스테이지 클리어 시 다음 스테이지가 해금되며, 진행도는 `PlayerPrefs`를 통해 저장됩니다.


### 2. 무한 모드

무한 모드에서는 랜덤한 퍼즐을 생성합니다.

단순히 무작위 보드를 생성하면 해결 불가능한 퍼즐이 나올 수 있기 때문에,  
생성된 퍼즐을 해 검증 알고리즘을 통해 검증한 뒤 해결 가능한 퍼즐만 게임에 사용합니다.


### 3. 해 검증 알고리즘

무한 모드의 핵심은 **랜덤으로 생성된 퍼즐이 실제로 해결 가능한 지 판별하는 것**입니다.

Lights Out 퍼즐은 각 블록을 누를지 말지를 '0' 또는 '1'로 표현할 수 있고,
각 입력이 주변 블록에 미치는 영향을 행렬로 나타낼 수 있습니다.

이 프로젝트에서는 다음 과정을 통해 퍼즐의 해결 가능 여부를 판단합니다.

'''text
현재 보드 상태 입력
        ↓
각 버튼이 영향을 주는 위치를 기준으로 계수 행렬 생성
        ↓
현재 보드 상태를 벡터로 변환
        ↓
확장 행렬 생성
        ↓
GF(2) 기반 가우스 소거법 수행
        ↓
모순 행 존재 여부 확인
        ↓
해결 가능 / 해결 불가능 판별
'''

### 알고리즘 적용 이유
무한 모드에서 단순 랜덤 생성만 사용할 경우,
플레이어가 아무리 조작해도 해결할 수 없는 퍼즐이 생성될 수 있습니다.

이를 방지하기 위해 생성된 보드에 대해 먼저 해 검증을 수행하고,
해결 가능한 보드만 실제 플레이에 사용하도록 구성했습니다.

## 데이터 관리

### 스테이지 데이터

스테이지 정보는 CSV 파일로 관리합니다.

```text
1,,,,,,
1,1,0,,,,
1,0,1,,,,
0,1,1,,,,
```

각 값은 다음 의미를 가집니다.

| 값 | 의미 |
| --- | --- |
| `0` | 꺼진 블록 |
| `1` | 켜진 블록 |

스테이지 데이터를 코드와 분리하여 관리함으로써,  
새로운 스테이지를 추가하거나 난이도를 조정할 때 코드 수정 없이 CSV만 수정할 수 있도록 했습니다.

---

### 플레이어 데이터 저장

플레이어의 진행도와 설정값은 `PlayerPrefs`를 통해 저장합니다.

| 데이터 | 설명 |
| --- | --- |
| 선택한 스테이지 | 현재 플레이할 스테이지 |
| 해금된 스테이지 | 클리어 진행도 |
| 현재 점수 | 무한 모드 현재 점수 |
| 최고 점수 | 무한 모드 최고 기록 |
| BGM 음소거 | 배경음 음소거 여부 |
| SFX 음소거 | 효과음 음소거 여부 |
| BGM 볼륨 | 배경음 볼륨 |
| SFX 볼륨 | 효과음 볼륨 |

## 기술적 고민

### 1. 해결 가능한 랜덤 퍼즐 생성

무한 모드에서 가장 중요하게 생각한 부분은  
랜덤 생성된 퍼즐이 플레이어에게 불공정하지 않아야 한다는 점이었습니다.

단순 랜덤 보드는 해결 불가능한 상태가 나올 수 있기 때문에,  
퍼즐을 행렬로 변환하고 가우스 소거법을 적용해 해가 존재하는지 검증했습니다.

이를 통해 무한 모드에서도 항상 해결 가능한 퍼즐만 제공할 수 있도록 했습니다.

---

### 2. 스테이지 데이터와 코드 분리

스테이지 정보를 코드 안에 직접 작성하면 스테이지를 수정할 때마다 코드를 변경해야 합니다.

이를 개선하기 위해 스테이지 정보를 CSV 파일로 분리했습니다.

이 방식의 장점은 다음과 같습니다.

- 코드 수정 없이 스테이지 추가 가능
- 난이도 조정이 쉬움
- 데이터 파싱 로직을 별도로 테스트 가능
- 게임 로직과 콘텐츠 데이터를 분리할 수 있음

---

### 3. 테스트 가능한 구조 만들기

Unity에서는 많은 기능이 `MonoBehaviour`와 씬 오브젝트에 의존하기 쉽습니다.

하지만 모든 로직이 `MonoBehaviour`에 묶이면 테스트가 어려워지기 때문에,  
가능한 로직은 일반 C# 클래스로 분리했습니다.

예를 들어 `LightsOutSolver`, `LevelLoader`, `DataService`는  
게임 오브젝트 없이도 테스트할 수 있도록 구성했습니다.
